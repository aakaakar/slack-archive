@sumitks - the state "database" - which is snapshot of the latest value(s) for the various keys - is going to basically be the same size as a normal database for the same number of records (key/value pairs). but along with that we also have the "ledger" (the closest analog in the database world would be the transaction log) which will contain the entire history of updates for every key. This will definitely be larger in size than a database log although by how much will depend on the number of endorsements required and a few other factors. I'll let the ledger folks add / correct me :wink:

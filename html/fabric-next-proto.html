aso
2016-10-13 08:51
@nikileshsa the visibility field controls to which extent the 'ChaincodeProposalInputs' (inside the original Proposal) is part of the final 'Transaction' message AND how is it included in the proposalHash field of the ProposalResponsePayload message. Imagine visibility accepts three values for the three functions you described: 1) if it's set to "identity function", the bytes of ChaincodeProposalInputs are stored in ChaincodeProposalPayload (in the ChaincodeActionsPayload field) and they are fully hashed into proposalHash (alongside the ProposalHeader); 2) if it's set to "hash function", H(ChaincodeProposalInputs) is stored in ChaincodeProposalPayload (in the ChaincodeActionsPayload field) and it is hashed into proposalHash (alongside the ProposalHeader); 3) if it's set to "empty byte array", then ChaincodeProposalPayload is empty and proposalHash only hashes the ProposalHeader;

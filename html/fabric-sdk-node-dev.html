@grapebaba Kai, I added the following explanation to the PR comment in response to your question: The main reason to remove typescript is to simplify the tool chain in the contributor's development environment. the original motivation for using TypeScript on the node.js SDK project revolved around it's affordance for code structure constructs such as interfaces and classes. while that can be easily achieved with much simpler prototype based javascript techniques as demonstrated in base.js. along with CommonJS modules we can have the equivalence of interfaces and classes rather easily. for strong typing, I don't think it's a big problem really.

A few words about events as an introduction for new comers benefit.

muralisr
2016-05-13 12:25
Fabric’s invoke and deploy requests are “submitted” and return an transaction ID. They are then executed asynchronously.

muralisr
2016-05-13 12:26
One can poll for completion of transaction or have an Event listener to learn about them.

muralisr
2016-05-13 12:27
Events do not send out “transaction” complete. They only send out “block” complete.

muralisr
2016-05-13 12:29
The only event _currently_ supported by Events is “block”. There is work on “Custom” or “Chaincode” events (https://github.com/hyperledger/fabric/wiki/Custom-Events-High-level-specification) where chaincode can emit events

muralisr
2016-05-13 12:32
There are sample for events which also serve as documentation currently. These are “fabric/events/events_test.go” and “fabric/examples/events/block-listener"

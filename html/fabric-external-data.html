jzhang
2016-05-24 13:27
so the general context goes like this: - a smart contract needs input from an external source (e.g price of a particular stock, bank balance of a particular account, at a given point in time) - by design smart contracts are prohibited from directly contacting outside world (no access to network, file system, etc.) - an intermediary of some sort is needed to supply this input to the SC execution - this intermediary must prove that the data it relays is identical to the data obtained from the external source at the given point in time - if some parts of the intermediary runs off of the blockchain network, the “proof” must include attestation of secured execution of code to guard against attacks on hardware and vm/os level

jzhang
2016-05-24 13:30
another piece of information is that most API providers don’t sign their output, as a result when the output passes through an intermediary, there’s no guarantee that it hasn’t been modified before it’s passed along to the requester

jzhang
2016-05-24 13:36
if the above sufficiently covers the requirements, then I think the Town Crier project addresses them pretty well

kostas
2016-05-24 13:37
curious: what's the drawback of the Oraclize/TLSnotary approach compared to Town Crier?

jzhang
2016-05-24 13:39
my understanding is that this requires a 2 step procedure: 1) the data is retrieved using a TLSnotary client and 2) oraclize proves that the data was retrieved using TLSnotary

jzhang
2016-05-24 14:10
so the proof is on data level, that it hasn’t been altered after it was obtained from the source. but i think there are other questions I don’t know if oraclize addresses, for instance have the request parameters been altered after it entered the intermediary (the oraclize server)? in other words i don’t believe there’s proof that the AWS servers hosting the service haven’t been tempered with, or if there’s no malicious os-level or application-level code that tries to modify the parameters.

jzhang
2016-05-24 14:59
another limitation of Oraclize is that it only supports public APIs

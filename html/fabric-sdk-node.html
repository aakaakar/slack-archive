@abhishekseth this is your key message: Error building images: Tag latest not found in repository http://docker.io/hyperledger/fabric-baseimage The only reason I can think of for this working with REST API but not with NodeSDK is if you were running in DEV (a.k.a. chaincodedev) mode with the REST API. Typical workflows I've seen have either: 1) pull down the appropriate tagged fabric-baseimage and tag it as latest, e.g. *docker tag hyperledger/fabric-baseimage:x86_64-0.2.2 hyperledger/fabric-baseimage:latest* or 2) tag a fabric-peer image as fabric-baseimage:latest (which may not be as 'elegant' but it usually works) e.g. *docker tag hyperledger/fabric-peer:latest hyperledger/fabric-baseimage:latest* Which way would work best for you may depend on how you got your images and what version you're using.
